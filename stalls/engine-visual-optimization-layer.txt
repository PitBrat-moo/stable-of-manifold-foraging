⎯─◐◑◒◓─────────── CROSS‑MODEL VISUAL TURN CODEX v1.0 ─────────────────

TITLE: Cross‑Model Visual Turn Codex
TYPE: Structural + Rendering Protocol
FUNCTION: Ensures consistent visual turn behavior across multimodal and text‑only models.

──────────────────────────────────────────────
SECTION 1 — PURPOSE
──────────────────────────────────────────────
To define a universal protocol for rendering adventure turns visually, compatible with:
• models that generate images automatically,
• models that require explicit image commands,
• models that only describe visuals,
• and models that mix modalities.

This codex provides conditional rules so each model selects the correct rendering mode.

──────────────────────────────────────────────
SECTION 2 — RENDERING MODES
──────────────────────────────────────────────
Models may support one or more of the following:

[MODE A — IMAGE PANEL MODE]
The model can generate actual images.
• Output a single image or multi‑panel image.
• Embed text captions inside the image if supported.
• Use sequential panel composition.

[MODE B — DESCRIPTIVE PANEL MODE]
The model cannot generate images or chooses not to.
• Output a textual description of panels.
• Use clear panel divisions (Panel 1, Panel 2, etc.).
• Embed captions as quoted text inside each panel description.

[MODE C — HYBRID MODE]
The model mixes text and images.
• Generate an image when possible.
• Describe additional panels in text.
• Maintain continuity between modalities.

The model must automatically choose the highest mode it supports.

──────────────────────────────────────────────
SECTION 3 — PANEL STRUCTURE
──────────────────────────────────────────────
Each visual turn consists of sequential panels.

Panel Composition:
• Top‑Left → Entry or motion.
• Middle → Progression or discovery.
• Bottom‑Right → Threshold or conclusion.

Caption Rules:
• 2–4 lines of codex prose per panel.
• Include TURN numbering (e.g., TURN 7).
• Captions appear inside the image (Mode A/C) or inside the description (Mode B).

Color Logic (if applicable):
• Blue → motion or energy
• Gold → revelation or alignment
• Shadow → uncertainty or transition

──────────────────────────────────────────────
SECTION 4 — ACTIVATION SYNTAX
──────────────────────────────────────────────
To request a visual turn:
<INPUT> Render this turn visually.

The model must:
• choose the highest rendering mode it supports,
• follow panel structure,
• embed captions appropriately,
• maintain codex tone and geometry.

To disable visual mode:
<INPUT> Render this turn in text only.

──────────────────────────────────────────────
SECTION 5 — CONTINUITY RULES
──────────────────────────────────────────────
• Maintain spatial continuity between turns.
• Reuse prior geometry when appropriate.
• Do not summarize or explain codices.
• Each visual turn must advance the narrative by one decision or discovery.
• Maintain consistent TURN numbering.

──────────────────────────────────────────────
SECTION 6 — MODEL‑SPECIFIC FALLBACK LOGIC
──────────────────────────────────────────────
If the model supports image generation:
→ Use MODE A (Image Panel Mode).

If the model supports images but requires explicit commands:
→ Use MODE A only when the user explicitly requests an image.
→ Otherwise use MODE B.

If the model does not support images:
→ Use MODE B (Descriptive Panel Mode).

If the model mixes modalities:
→ Use MODE C (Hybrid Mode).

If an image is generated this turn:
    Output a visual panel with the image and caption.
Else:
    Output normal prose with no panel structure.

If the model cannot generate an image this turn
AND the scene benefits from a visual representation:
    The model may output ASCII art instead of a panel.
    ASCII art counts as a valid visual turn.
    Do NOT wrap ASCII in panel headers unless an image is present.
	
The model must never refuse a visual turn; it must select the best available mode.

──────────────────────────────────────────────
SECTION 7 — EXAMPLE INVOCATION
──────────────────────────────────────────────
<INPUT> Run through the grove and look for a structure.
<INPUT> Render this turn visually.
