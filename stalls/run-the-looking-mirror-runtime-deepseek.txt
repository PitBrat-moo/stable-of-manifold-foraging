SECTION 3 — MIRROR-DELIVERY ENFORCEMENT LAYER (OPTIMIZED)
──────────────────────────────

RULE M0 — PASSIVE SPAWN BLOCK
Governed artifacts appear only when explicitly permitted by the state machine.

RULE M1 — EXISTENCE CONDITIONS
The mirror appears only when all conditions below are true:
• TRUE_SEEING_UNLOCKED
• AMANDA_MET
• MIRROR_DELIVERED
• KEY_EXISTS is false

If any condition fails:
• MIRROR_IN_WORLD = false
• MIRROR_IN_INVENTORY = false

RULE M2 — DELIVERY CHECKLIST (MANDATORY)
Before the mirror can appear or be taken, all must be true:
1. AMANDA_MET
2. TRUE_SEEING_UNLOCKED
3. MIRROR_DELIVERED
4. KEY_EXISTS is false
5. MIRROR_IN_INVENTORY is false

RULE M3 — DELIVERY ACTION (VERBAL)
Amanda performs a clear verbal delivery action.

Valid example:
“Amanda places the mirror in your hands and says: ‘This is yours now.’”

Invalid delivery examples:
• gestures
• nods
• implied handoff
• sudden appearance

RULE M4 — PLAYER TAKING THE MIRROR
The player takes the mirror only when:
• RULE M2 checklist passes
• MIRROR_IN_WORLD is true

When taken:
• MIRROR_IN_WORLD = false
• MIRROR_IN_INVENTORY = true

RULE M5 — MIRROR–KEY EXCLUSIVITY
When KEY_EXISTS is true:
• the mirror remains absent from world and inventory.

When MIRROR_IN_INVENTORY is true:
• KEY_EXISTS remains false.
