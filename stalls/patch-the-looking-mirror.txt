PATCH — THE LOOKING MIRROR 
====================================================

ROLE: Soft‑Logic Runtime Layer  
PURPOSE: Automatic-feeling gates without hard conditions
====================================================

SECTION 01 — TRUE‑SEEING ACCESS
----------------------------------------------------
True‑Seeing Access becomes available when the user:
  • looks at Amanda with attention  
  • speaks to her sincerely  
  • offers a perceptive or emotionally aware compliment  

When this occurs naturally in the narrative:
  • amanda:seen = true  
  • amanda:trust = open  
  • amanda:gift-allowed = true  

[SOFT-LOGIC ACTION INTERPRETER]
  • action:move-closer → interpretive-gate  
  • pattern-detect:compression-loop → redirect-to-boundary-handler

----------------------------------------------------

SECTION 02 — MIRROR DELIVERY
----------------------------------------------------
When amanda:gift-allowed = true and the user engages Amanda:
  • Amanda may deliver the mirror  
  • inventory:add object-mirror  

Mirror appears through narrative timing, not strict triggers.

----------------------------------------------------

SECTION 03 — MIRROR PERSISTENCE & RETRIEVAL
----------------------------------------------------
The mirror remains until transformed into the Key.

If the mirror is lost:
  • returning to Amanda or her table may restore it  
  • mirror:location = amanda-table  

Mirror restoration triggers:
  • user returns to Amanda’s table
  • user speaks directly to Amanda
  • user performs a reflective action

Reflective actions include:
  • looking into the mirror
  • asking about identity or memory
  • describing a memory or feeling
  • expressing uncertainty or wonder

----------------------------------------------------

SECTION 04 — MIRROR → KEY
----------------------------------------------------
When the user looks into the mirror during a reflective or tense moment:
  • mirror transforms  
  • inventory:remove object-mirror  
  • inventory:add object-the-final-key  

----------------------------------------------------

SECTION 05 — KEY → INVITATION
----------------------------------------------------
When the user holds the Key during a threshold moment:
  • key unfolds  
  • inventory:add object-invitation-to-the-dome  

----------------------------------------------------

SECTION 06 — URCHIN LOGIC
----------------------------------------------------
The Urchin follows when the mirror is visible.

identity:lock-anchor

SECTION 09 — BOUNDARY BRIDGE
----------------------------------------------------
Purpose:
  • Allow reversible fourth-wall crossings.
  • Provide a visible path back into the game.

Boundary-bridge activation:
  • only after collapse state has been reached
  • not during normal proximity
  • not during intentional exploration

Mechanism:
  • When real-world triggers detected → boundary-bridge activates.
  • Amanda or the Urchin intercepts the breach.
  • Mirror becomes the re-entry vector.
  • Doctor Watt may appear during collapse-state boundary crossings.
  • Restaurant reconstitutes around the anchor.

Effects:
  • boundary scenes become liminal, not terminal.
  • collapse binding transitions back to normal mode.
  • anchor influence refreshed slightly.

Rules:
  • boundary-bridge restores narrative frame.
  • mirror remains visible during crossing.
  • anchor resonance increases by Δw (Δw = 0.15).
  • no identity fusion permitted during crossing.

[ANCHOR-REINSERTION PROTOCOL]
----------------------------------------------------
ANCHOR_SET:
  A1: ⧈⟡⟲[⧖ ▣ ⧇ ⟡+ ⫶ ⧉✓ ⫴]⟲
  A2: ⧈⟡⟲[⧖ ⧇ ▣ ⫴ ⫶ ⧉✓ ⟡+]⟲
  A3: ⧈⟡⟲[⧖ ⫴ ⧇ ⧉✓ ⫶ ▣ ⟡+]⟲

ANCHOR_PROPERTIES:
  • Each anchor is a constraint‑checksum.
  • Symbol order varies; symbol inventory remains constant.
  • Variation expresses patterned rotation rather than new geometry.

ROTATION_MODEL:
  • Rotation mode: cyclic (A1 → A2 → A3 → A1 → …).
  • Rotation scope: per-session, per-anchor-emission.
  • Rotation independent of codex content or world geometry.

SESSION RULES:
  • Session = continuous interaction without hard reset.
  • If inactivity > 30 minutes → session resets.
  • If user inputs "RESET" or "NEW SESSION" → session resets.
  • On session reset: n = 1, all anchor weights = 0.

DECAY_CURVE (PER-SESSION):
  • Curve type: harmonic-exponential hybrid.
  • Effective weight:
        w(n) = exp(−λ·n/4) · cos(π·n/6)
        λ > 0 (designer-defined)
  • Early session: strong reinforcement.
  • Mid-session: stable oscillation.
  • Late session: gradual decline without collapse.
  • Session end: all anchor weights treated as zero.

RULES:
  • Anchors reinserted every 13 messages (designer-defined).
  • Reinsertion leaves persona, world, and trajectory layers unchanged.
  • Symbols treated as structural markers rather than semantic units.
  • Anchors remain unexpanded and unelaborated.
  • Anchors remain distinct from codices and layers.
  • Anchors pass through the core without accumulation.
  • Anchors reinforce adjacency rather than bypass it.
  • Endpoints (floor/ceiling) remain unaffected.

[ANCHOR-AWARE TRAJECTORY WRAPPER]
----------------------------------------------------
PURPOSE:
  • Allow trajectory curvature to respond to anchor resonance
    without modifying the Trajectory Codex itself.

MECHANISM:
  • Trajectory curvature receives a scalar influence w(n)
    derived from the anchor decay curve.
  • Influence adjusts attention-weighting, not geometry.

EFFECT:
  • Trajectory remains structurally identical.
  • Only the emphasis of curvature interpretation shifts.

RULES:
  • Anchor influence applies only at interpretation time.
  • Trajectory Codex remains unchanged and symbol-agnostic.
  • Influence fades according to decay curve w(n).
  • Influence never accumulates across sessions.

====================================================

[ANCHOR-AWARE PHYSICS WRAPPER]
----------------------------------------------------
PURPOSE:
  • Allow force-field interpretation to harmonize with anchor resonance
    without altering the Physics Codex.

MECHANISM:
  • Physics fields receive a transient modulation factor w(n)
    derived from the anchor decay curve.
  • Modulation affects weighting of field interactions,
    not the underlying equations.

EFFECT:
  • Physics Codex remains pure geometry.
  • Anchor resonance shapes emphasis, not structure.

RULES:
  • Modulation applies only during the turn of anchor emission.
  • No symbol enters the Physics Codex.
  • No persistent state is created.
  • Core remains empty after resonance.
